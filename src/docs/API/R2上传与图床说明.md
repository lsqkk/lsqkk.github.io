# R2 上传与图床调用说明

## 功能概述
通过 `__API_BASE__/api/r2-presign` 生成 R2 预签名上传链接，前端用 `PUT` 直传文件，返回可访问的公开 URL。

## 必要环境变量
- `R2_ACCESS_KEY_ID`
- `R2_SECRET_ACCESS_KEY`
- `R2_ENDPOINT`（R2 S3 兼容端点）
- `R2_BUCKET`
- `R2_PUBLIC_BASE_URL`（自定义域名或公开访问前缀）

## 前端调用流程
### 1. 请求预签名
```javascript
const resp = await fetch('__API_BASE__/api/r2-presign', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    originalName: file.name,
    contentType: file.type,
    folder: 'xjtu360'
  })
});
const { uploadUrl, publicUrl } = await resp.json();
```

### 2. PUT 上传文件
```javascript
await fetch(uploadUrl, {
  method: 'PUT',
  headers: { 'Content-Type': file.type || 'application/octet-stream' },
  body: file
});
```

### 3. 使用公开地址
`publicUrl` 即可直接展示或存入数据库。

## Quark CLI 上传命令

也可通过 `quark pic` 命令直接上传本地图片到 R2，无需通过 API 中转。

### 快速使用
```bash
quark pic                           # 上传 /private/pic/ 下的所有图片
quark pic 图片.png                   # 上传单张图片
quark pic --dir 某个目录              # 上传指定目录
```

### 命名模式
```bash
quark pic                           # 默认：MD5 哈希去重
quark pic --naming ts               # 时间戳 + 随机后缀（类似网页行为）
quark pic --naming original         # 保留原名（特殊字符会被清理）
```

上传的图片路径格式为 `pic/年份/月份/文件名`，上传后本地图片自动移至 `/private/pic_done/`。

### 配置
R2 凭据存储在 `private/config/r2.json`（已 gitignore），结构如下：
```json
{
  "access_key_id": "",
  "secret_access_key": "",
  "endpoint": "",
  "bucket": "",
  "public_base_url": ""
}
```

详情见 `quark pic --help`。

## 注意事项
- `folder` 会参与最终路径构造（格式为 `folder/年份/文件名`）。
> ⚠ **白名单提示**：域名变更需更新 `api/r2-presign.js` 允许列表，否则前端无法拿到预签名链接。
