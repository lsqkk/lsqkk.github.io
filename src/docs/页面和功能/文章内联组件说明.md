# 文章内联组件说明

文章 Markdown 中支持通过 HTML 标签语法嵌入特殊的交互组件，这些组件由 `astro.config.mjs` 中的自定义 rehype 插件在构建时转换为最终 HTML。

## 引用卡片 `<div id="card">`

**插件**：`rehypePostCard`（`astro.config.mjs`）

在 Markdown 中插入一个玻璃拟态风格的引用卡片：

```html
<div id="card" title="标题" intro="简介" tag="标签" description="副标签"
     href="链接" image="图片URL"></div>
```

### 属性说明

| 属性 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 固定为 `"card"`，用于触发插件识别 |
| `title` | 否 | 卡片标题（默认"未命名卡片"） |
| `intro` / `summary` | 否 | 卡片正文/简介 |
| `tag` | 否 | 标签徽章文字（accent 色） |
| `description` | 否 | 副标签（灰色） |
| `href` / `link` | 否 | 链接 URL，有则整个卡片可点击 |
| `image` / `img` / `src` | 否 | 缩略图 URL |

### 渲染效果

- 两列网格布局（148px 图 + 正文）
- 毛玻璃背景、圆角 22px
- 有链接时 hover 抬起 2px 效果
- 暗色模式自动适配

## 文件预览 `<filepreview>`

**插件**：`rehypeFilePreview`（`astro.config.mjs`，2026-06 新增）

在 Markdown 中嵌入文件预览组件，支持展开/折叠预览与下载：

```html
<filepreview src="/assets/pdf/xxx.pdf" title="显示名称" size="96KB"></filepreview>
```

### 属性说明

| 属性 | 必填 | 说明 |
|------|------|------|
| `src` | 是 | 文件路径（相对于站点根目录），文件需放在 `assets/` 下 |
| `title` | 否 | 显示名称（默认使用文件名） |
| `size` | 否 | 文件大小文本，如 `"96KB"`、`"2.1MB"` |

### 文件类型支持

| 类型 | 行为 |
|------|------|
| PDF | 使用浏览器内嵌预览（`<iframe>`），首次展开时懒加载 |
| 其他 | 显示"暂不支持在线预览，请下载后查看"提示，提供下载按钮 |

### 交互行为

- **折叠态**：默认显示紧凑文件信息卡片（图标 + 名称 + 大小 + 操作按钮）
- **展开预览**：点击卡片头部任意位置（下载按钮除外）展开内嵌预览
- **下载**：点击下载按钮直接下载源文件
- **键盘**：支持 Enter / Space 键展开/折叠
- **懒加载**：PDF 仅首次展开时加载，不浪费带宽

### 文件放置规则

- 源文件放在 `assets/` 下的对应目录，如 `assets/pdf/`、`assets/docs/`
- 路径以 `/assets/...` 引用，构建时 `scripts/sync-public.mjs` 会自动同步到 `public/assets/`
- 不要直接放在 `public/` 下（该目录为生成目录，不建议手动操作）

### 示例

```
<filepreview src="/assets/pdf/我们都很棒.pdf" title="B站2025毕业歌钢琴谱" size="96KB"></filepreview>
```

## 插件说明

两个插件均定义在 `astro.config.mjs` 中，通过 `markdown.rehypePlugins` 注册：

```js
rehypePlugins: [rehypeRaw, rehypePostCard, rehypeFilePreview],
```

- `rehypeRaw` 必须先于自定义插件，负责将 Markdown 中的原始 HTML 标签转换为 HAST 节点
- 自定义插件遍历 HAST 树，匹配特定标签/属性后替换为组件结构
- 文件预览插件的交互脚本由插件自动注入 HTML 末尾，无需手动引入 JS 文件

## 样式文件

- 引用卡片样式：`assets/css/post.css` 中 `.post-inline-card*` 规则
- 文件预览样式：`assets/css/post.css` 中 `.post-file-preview*` 规则
- 均使用全站统一的玻璃拟态设计变量（`var(--glass-bg)`、`var(--glass-edge)` 等）
