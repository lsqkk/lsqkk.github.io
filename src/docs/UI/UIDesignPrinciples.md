# 全站 UI 设计原则 (UI Design Principles)

## 概述

本文档记录了夸克博客全站统一的 UI 设计原则。这些原则源于对现有页面的提炼，以及经过多轮迭代验证的「什么效果好」的经验总结。新页面开发或旧页面重构时应优先遵守。

---

## 一、视觉基调

### 1.1 浊玉玻璃

全站以「浊玉」式毛玻璃为视觉基底，特征：
- **不追求全透明** —— `backdrop-filter: blur(18~22px)` 配合 `rgba(255,255,255,0.55)` 左右的背景，保留模糊但保持阅读舒适度
- **边缘有光** —— 使用 `inset 0 0 0 1px var(--glass-edge)` 模拟玻璃边缘高光
- **阴影柔和** —— 阴影使用 `var(--glass-shadow)`，颜色偏蓝灰而非纯黑，避免脏感

详情见 `GlassSystem.md`。

### 1.2 扁平结构优先

- 页面结构以**分区 + 留白 + 分隔线**组织，避免多层嵌套卡片
- 列表优先用分割线列表，不用卡片列表
- 小信息块（时间、标签、统计数字）直接放在内容流中，不独立成框
- 详情见 `FlatUI.md`

### 1.3 克制用色

- 主色调为蓝色系（`--accent: #2d6dc4`），辅助色为绿色系（`--accent-2: #1e8f6b`）
- 颜色有语义：蓝色=可交互/链接，绿色=成功，红色=错误/删除，黄色=阅读时间标签
- 不要在同一页面使用超过 3 种强调色

## 二、布局原则

### 2.1 内容宽度

| 页面类型 | 最大宽度 | 说明 |
|---------|---------|------|
| 阅读型（文章、动态） | `800px` | 保证行长舒适 |
| 列表型（工具、游戏） | `1200~1400px` | 网格自适应 |
| 主页 | `1280px` | 双栏布局 |
| 全屏型（404、登录） | 居中对齐 | 卡片式居中 |

### 2.2 间距体系

使用统一的间距（在 `tokens.css` 中定义）：

- 内容区域 padding：`20~24px`
- 卡片间距：`20~25px`
- 列表项间距：`12~16px`
- 元素内间距：`12~20px`

不要在同一页面混用多个间距体系（比如一边 `15px` 一边 `20px`）。

### 2.3 响应式断点

| 断点 | 变化 |
|------|------|
| `1120px` | 主页双栏 → 单栏 |
| `768px` | 侧栏堆叠、网格变为 1 列 |
| `640px` | 内边距缩减、字号调整 |

## 三、内容呈现原则

### 3.1 标题层级

- H1（页面标题）：`clamp(2rem, 6vw, 4.6rem)` 或等比例大字号
- H2（区块标题）：`1.2~1.8rem`，底部边框分隔
- H3（卡片标题）：`1.0~1.4rem`

区块标题下方有分隔线（`border-bottom`），分隔线颜色使用 `var(--line)`。

### 3.2 标签/徽章样式

- 圆角 `999px`（胶囊形）
- 不同语义使用不同背景色
- 字号 `0.74~0.85rem`
- 可交互的标签应当有 hover 反馈（抬起、变色、扩展）

### 3.3 卡片规范

- 圆角：`12px`（小）、`16px`（中）、`20~28px`（大）
- 背景：`var(--glass-bg)` 或 `var(--surface)`
- 边框：`1px solid var(--line)`
- 阴影：`var(--shadow-main)` 或 `var(--shadow-soft)`
- 内边距：`20~30px`
- 卡片悬停：抬起 `translateY(-2~-5px)` + 阴影加深

## 四、交互原则

### 4.1 动效体系

详见 `TransitionSystem.md`，核心要点：

| 交互类型 | 推荐动效 | 时长 |
|---------|---------|------|
| 悬停 | 抬起/变色/下划线展开 | `0.25~0.35s` |
| 聚焦 | 发光边框 + 微缩放 | `0.25~0.3s` |
| 按压 | `scale(0.96~0.98)` | `0.15s` |
| 入场 | 错位淡入 | `0.35~0.55s` |

所有动效必须支持 `prefers-reduced-motion` 和 `body.reduce-motion`。

### 4.2 加载反馈

详见 `SkeletonSystem.md`，核心要点：

- 所有异步加载内容的页面必须有骨架屏或加载指示器
- 骨架屏形态需贴近真实内容布局（标题行、文本行、图片位）
- 数据到达前显示骨架，到达后直接替换

### 4.3 表单交互

- 输入框聚焦：边框变色 + `box-shadow` 光晕 + 微缩放
- 按钮：悬停抬起 + 阴影增强，按压缩回
- 搜索框：聚焦时可适当扩展宽度
- 验证码/加载状态：在按钮旁或按钮内显示状态文字

### 4.4 文案规范

- 无数据时显示空状态提示（非空白）
- 错误状态显示可读的错误信息（非技术堆栈）
- 操作反馈有明确的状态行（`等待登录`、`保存成功`）

## 五、深色/浅色适配

### 5.1 CSS 变量驱动

使用 `tokens.css` 中的 CSS 变量实现双主题：

```css
.my-element {
    background: var(--glass-bg);
    color: var(--text-main);
    border-color: var(--line);
}
```

避免在页面 CSS 中硬编码颜色值。必须硬编码时，需同时提供 `body.dark-mode` 覆盖。

### 5.2 深色模式规则

- `@media (prefers-color-scheme: dark)` 自动适配
- `body.dark-mode` 手动切换（通过 `user-preferences.js`）
- 两个规则需要同步维护，不要遗漏

```css
/* 自动深色 */
@media (prefers-color-scheme: dark) {
    .my-element { color: #e2e8f0; }
}

/* 手动深色 */
body.dark-mode .my-element {
    color: #e2e8f0;
}
```

## 六、新页面开发检查清单

- [ ] 使用 `tokens.css` 的 CSS 变量，不硬编码颜色
- [ ] 毛玻璃使用 `var(--glass-bg)` 系列变量
- [ ] 深色模式已适配（`@media` + `body.dark-mode`）
- [ ] 动效已添加并支持 `reduce-motion`
- [ ] 异步数据加载有骨架屏
- [ ] 响应式断点已验证（768px, 640px）
- [ ] 空状态有提示文案
- [ ] 错误状态有友好提示
- [ ] 卡片/区块间距与全站一致

---

## 七、跨页面一致性要点

| 方面 | 统一方式 | 违反示例 |
|------|---------|---------|
| 卡片背景 | `var(--glass-bg)` | 某页用 `rgba(255,255,255,0.82)` 另一页用 `rgba(255,255,255,0.55)` |
| 卡片圆角 | `12px / 16px / 20px` | 某页 `8px` 另一页 `15px` |
| 边框色 | `var(--line)` | 直接写 `#dadce0` |
| 间距 | 参照间距体系 | 一处 `20px` 一处 `18px` |
| 标签样式 | 胶囊形 + hover 反馈 | 某页用方形标签 |
| 加载反馈 | 骨架屏或 spinner | 白屏无反馈 |
| 按钮交互 | 抬起 + 按压缩放 | 仅有颜色变化 |
