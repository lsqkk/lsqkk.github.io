# Flat UI 参考规范（无套娃扁平玻璃版）

本规范用于指导首页、聚合列表页、侧栏区块等页面的 UI 改造，目标不是机械套用某个数值，而是让后续 Agent 能直接理解这套视觉语言，做出和当前首页同一审美方向的效果。

这份文档优先描述“应当长什么样”和“绝对不要做成什么样”，数值只作为范围参考，不要求逐项照抄。

## 1. 核心目标

### 1.1 总体方向
- 视觉基调是“扁平玻璃 + 清晰结构 + 尽量少层级”。
- 页面可以有大面板，但不要出现“大框里再塞一排小框，小框里再塞评论小框”的套娃结构。
- 信息组织主要依赖：
  - 分区
  - 留白
  - 横向分隔线
  - 字号层级
  - 轻微背景差异
- 不依赖到处加卡片、描边、虚线、局部小容器来制造结构感。

### 1.2 一句话原则
- 一级结构可以有框，二级内容尽量不要再是框。
- 小信息块优先合并成一个完整区域，而不是一条信息一个盒子。
- 列表优先做“分隔线列表”，不是“卡片列表”。
- Hover 要轻、软、慢，不能突然亮一下或糊一层颜色。

## 2. 最重要的反模式

以下情况应优先视为需要重构：

### 2.1 套娃卡片
- 主栏有一个玻璃大框，里面文章列表每条又是玻璃小框。
- 动态列表每条是框，评论预览又再套一层框。
- 侧栏已经是一个大框，里面欢迎语、时间、IP 欢迎、统计又各自独立成多个小框。

### 2.2 过度碎片化
- 只包含一行字、一个 icon、一个时间的小内容，却单独占一个完整面板。
- 连续出现多个高度很小的块，导致页面像“模块拼图”。

### 2.3 假玻璃感过重
- 深色模式下左上角有明显白色高光渐变，像脏膜或没擦干净的塑料片。
- 浅色模式透明度太低，看起来像普通白卡片，不像玻璃。
- blur 很重，但层次不清楚，滚动时还拖慢渲染。

### 2.4 生硬交互
- hover 一上去立刻变蓝、变白、变亮。
- transform / background / border 没有渐变过渡。
- 动态这种信息流区域出现过强 hover 存在感，影响阅读。

## 3. 结构规则

### 3.1 允许存在的“框”
- 页面主栏的大面板。
- 侧栏的一组大面板。
- 页脚整体容器。
- 特殊用途的独立展示卡片：
  - 工具宫格
  - 项目卡片
  - 图片或媒体封面卡片
  - 明确需要点击的独立入口

### 3.2 不建议存在的“框”
- 文章列表里的每一篇文章。
- 首页动态流里的每一条动态。
- 留言预览里的每一条留言。
- 在线状态、友链、最近更新这种轻列表里的每一项。
- 评论预览里的每一条评论。

这些内容更适合做成：
- 同一个面板内的多条 row
- row 与 row 之间用横线分隔
- 配合轻微 hover 或完全不加 hover

### 3.3 小块合并规则
当某几个小块符合以下任意两条时，应考虑合并：
- 每块信息量很少。
- 高度很短。
- 语义上同属一个“欢迎/介绍/入口”区域。
- 拆开后并不能提升理解效率。
- 单独成块会让页面显得零碎。

常见合并方式：
- 主页主栏：头像 + 昵称 + 欢迎语 + 打字机 + 联系方式 + 社交链接，可合并为一个英雄引导区。
- 主页侧栏：问候语 + 当前时间 + IP 欢迎，可合并为一个欢迎信息区。
- 资料型内容：统计 + 一句说明 + 快捷入口，可合并为一个信息面板。

### 3.4 列表区的推荐结构
适用于：
- 文章
- 动态预览
- 留言预览
- 最近更新
- 在线状态
- 友链

推荐结构：
- 外层一个面板
- 标题区一行
- 内容区是连续 row
- row 与 row 之间使用 `border-top` 或伪元素分隔线
- 每个 row 内再用文本、缩略图、tag、时间来组织

不要写成：
- 外层大框
- 内层每条再套 `.card` / `.panel` / `.post-item`

## 4. 哪些元素不算“套娃框”

以下元素可以保留圆角、轻背景或小面积胶囊样式，它们不属于本规范要重点消灭的套娃：
- tag
- 徽标
- 按钮
- 头像
- 小图标底座
- 数字 badge
- 图片缩略图
- 小型操作按钮

判断标准不是“有没有圆角”，而是“它是不是又形成了一层新的卡片容器”。

## 5. 玻璃质感规范

### 5.1 浅色模式
- 应偏高透，能明显看出背景在后方，但前景内容仍清晰可读。
- 避免做成厚重白卡片。
- 背景建议为高透明浅色玻璃，例如：
  - `rgba(255, 255, 255, 0.28 ~ 0.42)`
  - 边框 `rgba(255, 255, 255, 0.32 ~ 0.48)`
- blur 保持克制，通常比传统拟态更轻。

浅色模式关键词：
- 轻
- 透
- 干净
- 不油腻

### 5.2 深色模式
- 参考站内文章列表页、工具页、其他常见 dark 页面那类暗色玻璃。
- 不要发灰发白，不要糊成一坨纯黑。
- 推荐方向是带一点冷色倾向的高透深玻璃，例如：
  - `rgba(17, 28, 46, 0.68)`
  - `rgba(18, 30, 50, 0.66 ~ 0.72)`
  - `rgba(19, 31, 52, 0.62 ~ 0.7)`
- 描边通常比背景再亮一点点，但不要过亮。

深色模式关键词：
- 冷
- 透
- 稳
- 不脏

### 5.3 明确禁止
- 深色模式明显的左上角白色高光渐变。
- 浅色模式大面积奶白蒙版。
- 用夸张渐变模拟“高级感”。
- 多层 box-shadow 叠到发脏。

### 5.4 推荐做法
- 用统一的基础底色 + 轻描边 + 少量 blur。
- 如果需要层次差异，优先调整透明度和边框，而不是额外叠一层高光。
- 同页面中的主面板、侧栏面板、页脚面板应属于同一家族，不要一个是玻璃、一个是实体、一个又是发光板。

## 6. Hover 与动态反馈规范

### 6.1 总原则
- 所有 hover 都应有过渡。
- 过渡速度偏柔和，避免“一碰就跳”。
- 背景、边框、阴影、位移变化都要尽量轻。

### 6.2 各类内容的 hover 强度

#### A. 强 hover
适用于：
- 明确点击入口
- 工具卡片
- 项目卡片
- 大按钮

可使用：
- 很轻的位移
- 很轻的边框提亮
- 轻微背景变化

#### B. 弱 hover
适用于：
- 友链
- 在线成员
- 最近更新
- 普通文章行

可使用：
- 文字颜色轻微变化
- 行背景轻微提亮
- 微弱阴影或透明度变化

#### C. 无 hover 或极弱 hover
适用于：
- 动态流
- 评论预览
- 偏阅读型的信息流

尤其首页动态：
- 不应出现蓝色渐变 hover
- 不应出现一层突兀的亮色遮罩
- 不应因为 hover 破坏阅读节奏

## 7. 首页专用落地规范

首页是这套 Flat UI 的主要样板，后续改同类页面应优先参考这里。

### 7.1 首页主栏
- 主栏可以有少量几个大区块，但不要碎。
- 顶部介绍区域应尽量合并，形成一个完整的“个人引导面板”。
- 文章区应使用分隔线列表，不要继续沿用通用 `.post-item` 玻璃卡片。
- 动态区应是信息流，不是卡片流。

### 7.2 首页侧栏
- 欢迎语、时间、IP 欢迎这类轻信息应合并。
- 在线、友链、留言、新视频、GitHub 动态等适合用“标题 + row 列表”。
- 不要让侧栏堆满一排高度很短的小框。

### 7.3 页脚
- 页脚应视作页面收尾的大面板。
- 圆角、裁切、背景、描边要统一，避免出现边角漏底或圆角错位。

### 7.4 需要特别留意的继承问题
首页容易被全站通用样式误伤，改首页时应检查是否有以下类名在偷偷重新加框：
- `.post-item`
- `.main-content`
- `.sidebar-index`
- 其他全局 `.card` / `.panel` / `.glass-*`

如果首页要做 Flat 化，必要时应在首页专属样式中显式覆盖这些通用外观。

## 8. 分隔线列表的推荐写法

### 8.1 正确方向
```css
.section-list {
  display: flex;
  flex-direction: column;
}

.section-row {
  padding: 1rem 0;
  transition:
    background-color 0.24s ease,
    color 0.24s ease,
    border-color 0.24s ease,
    transform 0.24s ease;
}

.section-row + .section-row {
  border-top: 1px solid rgba(255, 255, 255, 0.16);
}
```

特点：
- row 自身不是卡片
- 层次来自节奏和分隔
- hover 可以非常轻，甚至不加

### 8.2 错误方向
```css
.section-card {
  padding: 1rem;
  border-radius: 18px;
  background: rgba(255, 255, 255, 0.22);
  backdrop-filter: blur(24px);
}

.section-card + .section-card {
  margin-top: 0.75rem;
}
```

这种写法很容易把“列表”做成“大框里一堆小框”，视觉上马上开始套娃。

## 9. 间距与节奏建议

### 9.1 面板内部
- 当多个小模块被合并进一个大面板后，优先用上下留白来分层，不急着加虚线。
- 只有当内容确实是“多条并列条目”时，才使用分隔线。
- 头像区、欢迎文案区、社交入口区之间，可先尝试用 margin/padding 组织，而不是先加线。

### 9.2 页面层级
- 一级面板之间留白可以明显一些。
- 同一面板内部 row 与 row 之间留白略紧。
- 不要让每一层的圆角、间距、阴影都一样，否则会像模板堆砌。

## 10. 性能与体验注意事项

### 10.1 不要为了玻璃感滥用 blur
- blur 越重，移动端和弱设备越容易卡。
- 侧栏若还有 `sticky`，又叠加强 blur，滚动和加载期更容易卡顿。

### 10.2 首页优先稳定
- 首页首屏内容多，尽量少做昂贵视觉特效。
- 如果一个效果“有一点好看，但明显更卡”，首页应优先砍掉。

### 10.3 动画节奏
- 页面载入动画可以有，但应短、轻、顺。
- hover 动画必须是辅助，不应喧宾夺主。

## 11. Agent 实施清单

后续 Agent 在按这份文档改 UI 时，至少要自检以下内容：

### 11.1 结构
- 是否仍然存在“大框里套小框”的列表？
- 是否有多个只装一两行内容的小面板可以合并？
- 文章、动态、留言、友链等是否已经尽量改成分隔线列表？

### 11.2 样式
- 浅色模式是否足够透？
- 深色模式是否接近站内常见 dark 卡片，而不是另起炉灶？
- 是否还有明显左上角高光渐变？
- 是否还有突兀的蓝色 hover、白色遮罩、突然提亮？

### 11.3 继承
- 是否被全局 `.post-item`、`.main-content`、`.sidebar-index` 等样式重新套回去了？
- 是否页面主体背后还残留一层没删干净的玻璃壳？

### 11.4 体验
- 未完全加载时滚动是否卡顿？
- 深色和浅色都是否完整检查过？
- 桌面端与移动端是否都看过？

## 12. 给后续 Agent 的执行建议

如果用户说：
- “别套娃”
- “框太多”
- “文章改成分隔线”
- “深色模式像文章列表页那种”
- “hover 太生硬”

通常应理解为：

1. 减少独立面板数量，合并碎块。
2. 把列表项从卡片改为 row + divider。
3. 去掉多余渐变高光，尤其深色模式。
4. 把 hover 改成更轻的透明度/边框/文字变化。
5. 检查是否有全局卡片样式泄漏到当前页面。

如果某个元素只是：
- tag
- badge
- 小按钮
- 缩略图
- icon 圆底

通常不用为了“去框”而把它们也全砍掉。

## 13. 本规范的适用范围

优先适用于：
- 首页
- 侧栏聚合区
- 文章列表类页面
- 动态列表类页面
- 留言/友链/在线成员等轻信息列表

不强制套用到：
- 工具宫格
- 游戏入口
- 项目展示卡片
- 强视觉运营页

这些页面仍可保留卡片化设计，但也应避免无意义套娃。
